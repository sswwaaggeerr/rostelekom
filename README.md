# RTK Brigade Planner (FastAPI + HTML/JS)

MVP для загрузки Excel, проверки данных и планирования задач бригад с учетом:
- обычные бригады: 08:00-17:00, обед 12:00-13:00;
- дежурная бригада: 08:00-20:00, обед 14:00-15:00;
- целевой процент выполнения задач: не менее 82%;
- разрешенный овертайм для части бригад (настраивается в минутах);
- **НОВАЯ**: кластеризация задач по территории;
- **НОВАЯ**: динамическое обновление расписания;
- **НОВАЯ**: защита от конфликтов (несколько бригад в одном месте);
- **НОВАЯ**: проактивные звонки клиентам;
- **НОВАЯ**: статистика загруженности районов.

## Запуск

```bash
pip install -r requirements.txt
uvicorn app.main:app --reload
```

Открыть: <http://127.0.0.1:8000>

## Поддерживаемые форматы Excel

### Формат 1: Стандартный (английские колонки)
- `task_id`, `address`, `visit_time`, `sla_deadline`, `task_type`, `client_type`, `duration_min`

### Формат 2: Русский (CRM Ростелеком)
- `Задание №` или `Л/С` → task_id
- `Адрес` → address
- `Дата и срок задания м.` → visit_time (формат: "22.04.2026 09:00 - 21:00")
- `Срок задания м.` или `Срок заявки м.` → sla_deadline
- `Тип задания` → task_type (Инсталляция/Снятие/Обследование/Устранение неисправности/Дополнительные работы)
- `Тип клиента` → client_type
- `Состояние задания` → status (Назначена исполнителю/Отложена)

### Опциональные колонки
- `lat` / `Широта` — координаты
- `lon` / `Долгота` — координаты

## Автоматическое преобразование типов задач

| В файле | В системе |
|---------|-----------|
| Инсталляция на площадке абонента | инсталляция |
| Снятие | снятие |
| Обследование / Осмотр | обследование |
| Устранение неисправности | техподдержка |
| Дополнительные работы | доработка |

## Автоматическое преобразование статусов

| В файле | В системе |
|---------|-----------|
| Назначена исполнителю | assigned |
| Отложена | postponed |
| Выполнена / Завершена | completed |
| Запланирована | scheduled |

## Новые возможности

### 1. Кластеризация по территории
Задачи группируются по географическому признаку (радиус по умолчанию 2 км). Все задачи в одном кластере назначаются одной бригаде.

### 2. Динамическое обновление
Можно загружать новые задачи в течение дня. Опция "Сохранить расписание" позволяет продолжить работу с текущими назначениями.

### 3. Загруженность районов
Система показывает статистику по кластерам: количество задач, назначенная бригада, просроченные задачи.

### 4. Проактивные звонки
Если у бригады есть свободное время и рядом есть задача на будущее время, система предлагает позвонить клиенту.

### 5. Маршрутизация по дорогам
Используется OSRM/ORS/Yandex для построения реальных маршрутов по дорогам (не прямыми линиями).

## Настройка API ключей (опционально)

Для улучшенной маршрутизации с учетом пробок добавьте в переменные окружения:

```bash
export ORS_API_KEY="your_openrouteservice_key"
export YANDEX_ROUTING_API_KEY="your_yandex_key"
```

Без ключей используется публичный OSRM сервер.
